# Bank Management System (Sistema de Gerenciamento Bancário)

Uma aplicação desktop completa de um **Sistema Bancário**, desenvolvida em **Java** utilizando interface gráfica nativa (**Swing / JFrame**). Este projeto foi construído acompanhando uma série prática de desenvolvimento passo a passo, cobrindo desde a arquitetura inicial até a persistência de dados e telas dinâmicas.

---

## Sobre o Projeto

O **Bank Management System** simula as operações fundamentais de um banco, permitindo o cadastro de clientes, autenticação segura, operações de caixa eletrônico (ATM) como saques, depósitos, consulta de saldo e extratos detalhados.

O objetivo principal deste projeto foi consolidar conceitos de **Programação Orientada a Objetos (POO)** em Java e explorar na prática a criação de interfaces ricas com a biblioteca gráfica Swing.

---

## Tecnologias e Ferramentas Utilizadas

- **Linguagem:** Java (JDK 8+)
- **Interface Gráfica (GUI):** Java Swing (`JFrame`, `JLabel`, `JTextField`, `JButton`, etc.)
- **Banco de Dados / Persistência:** MySQL / JDBC *

---

## Funcionalidades do Sistema

-  **Cadastro e Abertura de Conta:** Formulário completo para registro de dados pessoais e bancários do novo cliente.
-  **Autenticação Segura (Login):** Validação de número de conta/cartão e PIN (senha).
-  **Depósitos e Saques:** Operações de movimentação de saldo com validação de limites.
-  **Módulo ATM (Caixa Eletrônico):** Interface interativa simulando um terminal bancário real.
-  **Extrato de Conta (Mini Statement):** Exibição do histórico das últimas transações.
-  **Alteração de PIN:** Funcionalidade para redefinição segura de senha do cartão.

---

##  Aprendizados e Destaques Técnicos

Durante o desenvolvimento deste projeto, foram trabalhados os seguintes pontos:
- **Interfaces Gráficas (GUI):** Criação e manipulação de janelas com `JFrame`, gerenciamento de layouts, eventos de botões (`ActionListener`) e estilização de componentes visuais.
- **Programação Orientada a Objetos:** Aplicação prática de encapsulamento, separação de responsabilidades e reutilização de código.
